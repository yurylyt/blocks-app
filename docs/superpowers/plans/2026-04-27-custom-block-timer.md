# Custom Block Timer Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Allow starting the Pomodoro-style timer with a freeform "custom" name (no activity), in both the web app and the macOS menubar app.

**Architecture:** Make `running_timers.activity_id` nullable and add a `name` column with a CHECK constraint enforcing exactly-one-of (mirrors the existing pattern in `entries`). API accepts either `activityId` or `name`. Both UIs gain a "Custom…" affordance in the start dropdown that mirrors the existing `AddEntryMenu.vue` pattern. On half-completion, the inserted `entries` row carries either `activityId` or `name` exactly as the timer did.

**Tech Stack:** Nuxt 4 + Drizzle ORM (postgres-js) on the web; SwiftUI / `@Observable` on macOS. No test runner is configured for this project (per `CLAUDE.md`) — verification is `pnpm typecheck` + `pnpm lint` + manual testing in the dev server, plus Xcode build/run for the macOS app.

**Spec:** `docs/superpowers/specs/2026-04-27-custom-block-timer-design.md`

## Task 2: Server — `POST /api/timer/start` accepts `activityId` OR `name`

**Files:**
- Modify: `server/api/timer/start.post.ts`

- [ ] **Step 1: Replace the handler body**

Overwrite `server/api/timer/start.post.ts` with:

```ts
import { and, eq } from 'drizzle-orm'
import { useDb, schema } from '~~/server/database/client'

export default defineEventHandler(async (event) => {
  const userId = await requireUserId(event)
  const body = await readBody<{ activityId?: number, name?: string, startedDate?: string }>(event)

  const startedDate = String(body?.startedDate || '')
  if (!/^\d{4}-\d{2}-\d{2}$/.test(startedDate)) {
    throw createError({ statusCode: 400, message: 'Invalid startedDate' })
  }

  const hasActivityId = body?.activityId != null && Number.isFinite(Number(body.activityId))
  const freeformName = body?.name?.trim()
  if (hasActivityId === Boolean(freeformName)) {
    throw createError({ statusCode: 400, message: 'Provide exactly one of activityId or name' })
  }

  const db = useDb()

  const [existing] = await db.select({ id: schema.runningTimers.id }).from(schema.runningTimers)
    .where(eq(schema.runningTimers.userId, userId))
  if (existing) throw createError({ statusCode: 409, message: 'Timer already running' })

  let activityId: number | null = null
  let name: string | null = null

  if (hasActivityId) {
    activityId = Number(body!.activityId)
    const [activity] = await db.select({ id: schema.activities.id }).from(schema.activities)
      .where(and(eq(schema.activities.id, activityId), eq(schema.activities.userId, userId)))
    if (!activity) throw createError({ statusCode: 404, message: 'Activity not found' })
  } else {
    name = freeformName!
  }

  const [row] = await db.insert(schema.runningTimers).values({
    userId,
    activityId,
    name,
    startedDate,
    half: 1
  }).returning()

  return row
})
```

Notes:
- The `hasActivityId === Boolean(freeformName)` test is the XOR check: it errors if both are truthy or both are falsy.
- Existing-timer check happens before activity lookup so we don't burn a query when a timer is already running.
